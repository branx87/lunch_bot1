# Быстрый старт системы бекапов

## Минимальная настройка (5 минут)

### 1. Получите токен Яндекс.Диска

1. Откройте: https://yandex.ru/dev/disk/poligon/
2. Войдите в аккаунт Яндекс
3. Нажмите "Получить OAuth-токен"
4. Скопируйте полученный токен

### 2. Добавьте токен в .env

Откройте файл `data/configs/.env` и добавьте:

```env
# Настройки резервного копирования
YANDEX_DISK_TOKEN=y0_AgAAAA...ваш_токен_здесь...
YANDEX_DISK_FOLDER=/lunch_bot_backups
BACKUP_MONTHS=3
```

### 3. Перезапустите бота

```bash
docker-compose down
docker-compose up -d --build
```

### 4. Проверьте работу

Отправьте боту команду:

```
/backup_status
```

Вы должны увидеть:
- ☁️ Яндекс.Диск: подключен

### 5. Создайте тестовый бекап

```
/backup
```

Через 1-3 минуты вы получите подтверждение создания бекапа.

## Что дальше?

- Бекапы будут создаваться **автоматически каждую ночь в 03:00**
- На Яндекс.Диске будут храниться **7 файлов** (по одному на каждый день недели)
- Локально будут храниться все бекапы (старше 7 дней удаляются автоматически)
- В бекап включаются только данные **за последние 3 месяца**

## Команды администратора

- `/backup` - создать резервную копию сейчас
- `/backup_status` - посмотреть статус бекапов
- `/backup --local` - создать только локальную копию (без облака)

## Восстановление из бекапа

См. подробную инструкцию в файле [BACKUP_SETUP.md](BACKUP_SETUP.md#восстановление-из-бекапа)

## Что входит в бекап?

- ✅ Все пользователи и сотрудники
- ✅ Меню и праздники
- ✅ Заказы за последние 3 месяца
- ✅ Сообщения за последние 3 месяца
- ✅ Настройки бота

**Размер бекапа:** обычно 1-5 МБ в сжатом виде
